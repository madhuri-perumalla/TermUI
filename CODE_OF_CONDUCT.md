# Code of conduct

## The short version

Be decent to each other. We're building terminal UI stuff, not running a debate club. Disagreement about code is fine. Personal attacks aren't.

## The longer version

This project is open to anyone regardless of experience level, background, or identity. We want contributions from people who have shipped terminal apps and from people writing their first TypeScript file.

### Expected behavior

- **Be constructive in code reviews.** "This approach has an issue because X, here's an alternative" is useful. "This is wrong" without explanation is not.
- **Ask questions.** If something in the codebase doesn't make sense, open an issue. The architecture has 14 packages — nobody expects you to understand all of it immediately.
- **Give credit.** If your PR builds on someone's idea from an issue or discussion, mention it.
- **Be patient with response times.** This is an open-source project maintained by humans with other commitments.

### Not acceptable

- Harassment, insults, or personal attacks
- Publishing others' private information
- Trolling or deliberately inflammatory comments
- Sustained disruption of discussions

### Enforcement

If someone violates this code of conduct, maintainers will:

1. Give a warning and explain what was wrong.
2. Remove the offending content if necessary.
3. Temporarily or permanently ban repeat offenders.

Reports of unacceptable behavior can be sent to **karanjotsingh786@gmail.com**. All reports are reviewed and handled confidentially.
