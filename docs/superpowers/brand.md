# TermUI Brand and Positioning

The single source of truth for how to describe TermUI. Read before writing any
marketing copy, docs intro, meta tag, OG card, or AI-citation surface.

## One-line positioning

> TermUI is a TypeScript framework for building terminal apps.

Not "terminal UIs." Not "a UI library." TermUI is a full application framework.
The UI layer is one part of it.

## Why "apps", not "UIs"

TermUI ships an app stack, not only a renderer. Verified from the 15 packages:

- UI layer: core (screen, layout, input), widgets, ui, jsx runtime, tss theming, motion
- App layer: store (global state), router (screen routing + nav guards), data
  (system providers: CPU, memory, disk, network, processes), adapters (git, AI,
  shell, github, keychain), dev-server (hot reload), testing (test renderer),
  quick (fluent builder), cli (component installer)

Routing, state, data, dev tooling, and testing make it a framework, not a widget set.

## The hook

> Build terminal apps in TypeScript. No React.

The "No React" line is the strongest differentiator. Ink, the default answer in
this space, depends on React and Yoga. TermUI ships its own JSX runtime. Lead
with this in comparison and Ink-alternative contexts.

## Ground-truth facts (keep these exact)

- 15 published packages (14 @termuijs/* + create-termui-app).
- 230 components.
- Version 0.1.6.
- MIT license.
- Node.js 18+ for published packages. Bun 1.3+ only for the dev server.
- Zero C extensions. No React dependency. Own JSX runtime.
- 12 built-in TSS themes.
- The website runs on Next.js 16 (not TanStack Start).

When any of these change, update this file and search the site for the old number.

## Keyword strategy

Lead with "terminal apps" in taglines and titles. Keep these as secondary
keywords (they are real search terms, do not delete them):

- terminal UI framework
- TUI
- Ink alternative
- CLI framework TypeScript
- React for terminal

Defining "TUI" as "terminal user interface" is correct and stays. The rule is
about positioning copy, not the acronym's definition.

## Voice

- Spartan and factual. Short active sentences. Address the reader as "you".
- No em dashes. Use periods or commas.
- No marketing filler: avoid powerful, seamless, robust, blazing, effortless,
  game-changer, unlock, revolutionize, supercharge.
- Back claims with numbers (230 components, 15 packages, 1,000,000 rows).
